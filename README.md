
#  LazySequence: Движок ленивых вычислений с ординальной индексацией

Этот проект реализует концепцию **бесконечных ленивых последовательностей** на C++, используя математический аппарат **ординальных чисел** (трансфинитных индексов). Система позволяет генерировать значения по требованию, модифицировать структуру последовательности (вставка, удаление, конкатенация) и безопасно управлять памятью благодаря паттернам глубокого копирования и мемоизации.

---

## Математическая основа

В отличие от обычных массивов, индексируемых натуральными числами ($0, 1, 2, \dots$), данная система использует **ординальные индексы** вида $n\omega + k$, где:
*   $\omega$ (омега) — первый трансфинитный ординал, представляющий предел бесконечной последовательности.
*   $n$ (`omegaPart`) — коэффициент при омеге (номер "блока" бесконечности).
*   $k$ (`finitePart`) — конечное смещение внутри этого блока.

**Ключевое правило ординальной арифметики, заложенное в архитектуру:**  
Сложение ординалов **не коммутативно**. Например, $1 + \omega = \omega$.  
Это означает, что добавление конечного числа элементов в начало бесконечной последовательности (`Prepend`) сдвигает только конечные индексы, но **не сдвигает** границу $\omega$ и все последующие блоки. Эта логика строго соблюдается в методах сдвига (`ShiftAllRanges`, `ShiftCachedAfter`).

## Описание внутренних модулей

### 1. Ядро индексации
*   **`OrdinalIndex`**: Структура, хранящая `omegaPart` и `finitePart`. Реализует все операторы сравнения (`<`, `>`, `==`) и математические операции сдвига (`ShiftByOmega` для вставки бесконечных блоков, `ShiftBy` для конкатенации).
*   **`OrdinalIndexParser`**: Утилита для парсинга строковых представлений индексов (поддерживает форматы `5`, `w`, `W`, `ω`, `2w+10`).

### 2. Генерация значений
*   **`IGenerator<T>`**: Абстрактный интерфейс. Знает только о *локальных* индексах ($0, 1, 2\dots$). Содержит метод `Clone()`, критически важный для безопасного глубокого копирования.
*   **`RuleGenerator<T>`**: Базовая реализация, применяющая функцию `T (*)(int)` к локальному индексу.

### 3. Управление диапазонами (Сердце системы)
*   **`GeneratorRange<T>`**: Структура, связывающая ординальный интервал `[start, end)` с конкретным `IGenerator` и хранящая `localOffset` (смещение, необходимое при разрезании генератора).
*   **`CompositeGenerator<T>`**: Управляет массивом `GeneratorRange` и массивом удаленных индексов (`removedIndices`). 
    *   **Расщепление (Splitting):** При `InsertSequence` в середину существующего диапазона, он аккуратно разрезается на две части. Правая часть сдвигается на новый уровень омеги с корректным пересчетом `localOffset`.
    *   **Удаление:** Логическое удаление запоминает *истинный локальный индекс*, чтобы при последующей генерации корректно пропускать удаленные элементы.

### 4. Публичный интерфейс
*   **`LazySequence<T>`**: Наследуется от базового `Sequence<T>`. 
    *   Адаптирует конечный интерфейс `Sequence` (с `int` индексами) к бесконечной ординальной логике.
    *   Реализует **мемоизацию**: метод `Get(OrdinalIndex)` при первом обращении генерирует значение и сохраняет его в `cache`, чтобы вернуть безопасную ссылку `const T&`.
    *   Предоставляет как мутирующие методы (`PrependMutate`), так и иммутабельные (`PrependLazy`), возвращающие новые объекты.
    *   Честно выбрасывает `BaseException` для операций, не имеющих смысла для бесконечных множеств (`GetLast`, `Map`, `Reduce`).

## Сборка и запуск

Для компиляции требуется компилятор с поддержкой **C++17** (GCC, Clang или MSVC).

### 1. Компиляция интерактивной демо-версии
```bash
g++ -std=c++17 -I. -Ibase main.cpp -o lazy_demo
./lazy_demo
```
*В демо-версии доступна история версий (до 10 штук), позволяющая отменять изменения и переключаться между состояниями последовательности.*

### 2. Компиляция и запуск модульных тестов
```bash
g++ -std=c++17 -I. -Ibase test_lazy_sequence.cpp -o test_lazy
./test_lazy
```
*Тесты покрывают: базовую генерацию, иммутабельность, ординальную арифметику при удалении, расщепление генераторов, конкатенацию со сдвигом и безопасность глубокого копирования.*
